Documentation on online examination portal

Introduction

Online Examination System is a web application that establishes a network between the websites and the students. The website has their own particular sets of questions. These questions are displayed as a test to the eligible students. The answers enter by the students are then evaluated and their score is calculated and saved. This score then can be accessed by the students to determine the passes students or to evaluate their performance.

Online examination System is being launched because of the need for a destination that is beneficial for both students and institutes. With this site, Students can register themselves to create an individual login IDs . Using this IDs, they can appear for examination on various computer science topics. This will help them to brush up their skills and they will be evaluated according to their performance which is generated by the system after completion of any particular test .

Online examination System fulfills the requirements of the institutes to conduct the exams online. They do not have to go to any software developer to make a separate site for being able to conduct exams online. They just have to register on the site and enter the exam details and the lists of the students which can appear in the exam. Students can give exam without the need of going to any physical destination. They can view the result at the same time. Thus the purpose of the site is to provide a system that saves the efforts and time of both the institutes and the students. Online Examination System provides the platform for the students to evaluate their knowledge on various topics of computer science.

The site has an administrator who keeps an eye on the overall functioning of the system. The site gets revenue by charging the students each time they want to participate in an exam. It is an application software which aims to improve the performance of students and thus making them more capable and aware about their subject , they are dealing with . It is developed by using ASP.NET using C# and related database.

Database Architecture

The design of a DBMS depends on its architecture. It can be centralized or decentralized or hierarchical. The architecture of a DBMS can be seen as either single tier or multi-tier. An n-tier architecture divides the whole system into related but independent n modules, which can be independently modified, altered, changed, or replaced. In 1-tier architecture, the DBMS is the only entity where the user directly sits on the DBMS and uses it. Any changes done here will directly be done on the DBMS itself. It does not provide handy tools for end-users. Database designers and programmers normally prefer to use single-tier architecture.

If the architecture of DBMS is 2-tier, then it must have an application through which the DBMS can be accessed. Programmers use 2-tier architecture where they access the DBMS by means of an application. Here the application tier is entirely independent of the database in terms of operation, design, and programming. Database Architecture of this online examination system is made of two different types of database, Authentication database and student database . The Authentication Database contains the name, email and password(hashed). It has the email as the primary key and it verifies the email which has been registered. If the email once registered is used up for logging in , the corresponding user will be allowed to use the system . Otherwise, an error page will show up to the user.

The Student Database contains the Student full name ,email and Score. It keeps the information of the students who appeared for any of the test and also keeps the scores in the database for evaluation of the student and help him decide if he is satisfied with his performance or if he has to improve it . The database architecture can be represented in a simple ER diagram in which User, Registration, Examination and Result are the entities and are joined to each other by relationship such as Register, gives , result, receive. Each of the entities have their own set of attributes and there is one multivalued attribute Subject.As of now in this software only the ASP.NET portion is implemented and available. The other topics will be implemented later on.

What are the user's demonstrable needs?

User needs a web-based system, which will remove all the above-mentioned Problems that, the user is facing. The user wants a web-based system, which will reduce the bulk of paperwork, provide ease of work, flexibility, fast record finding, modifying, adding, removing and generating the reports.


How can the problem be redefined?

We proposed our perception of the system, in accordance with the problems of existing system by making a full layout of the system on paper. Cost

The cost required in the proposed system is comparatively less to the existing system.

Effort

Time

Also the time required generating a report or for doing any other work will be comparatively very less than in the existing system. Record finding and updating will take less time than the existing system.

In the existing system the number of staff required for completing the work is more while the new system will require quite less number of staff.
